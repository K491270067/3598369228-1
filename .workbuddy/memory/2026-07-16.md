## 宏愿 UI 异常排查（2026-07-16 11:3x）
- 用户开启 TGP 实跑后报告「宏愿 UI 有异常」。调度 engineering-lead 查原版文件，结论已写入 `docs/house_aspiration_ui_analysis.md`。
- **根因（已二次核对原版代码确认）**：驱动家族集团的 6 个宏愿（ceremony/determination/humility/prosperity/service/strength，定义于 `common/house_aspirations/10_tgp_japan_house_aspirations.txt`，每 key 的 `is_shown = { house_head ?= { government_is_japanese_trigger = yes } }` 且 `confederation_type = house_bloc_*`）。`government_is_japanese_trigger`（`common/scripted_triggers/10_tgp_japan_triggers.txt:366-371`）= 仅认 `government_is_japan_administrative/feudal`。feudal_admin 的 flags 是 `government_is_feudal_admin/feudal`，**不含日本 flag** → 这 6 个宏愿对封建行政制全被排除。

- **关键判定（重要）**：这不是 GUI 问题。所有相关 GUI（`window_house_aspiration.gui`/`window_confederation.gui`/`window_dynasty_house.gui`）均数据驱动，靠 C++ 函数 `HasSpecialHouseAspiration`/`DynastyHouse.UsesHouseAspirations`/`GetAvailableHouseAspirations` 与 `is_shown` 决定；`IsHouseBasedConfederation` 来自联盟类型数据而非政府形态。**无需改 GUI**。

## 宏愿 GUI 布局强切为日本同款（2026-07-16 13:0x）
- 用户反馈：feudal_admin 数据走的是日本家族宏愿，但显示的 GUI 仍是行政制（administrative）的宏愿界面。目标：让此政体宏愿 GUI 调用日本同款（custom）布局。
